  <action>
Rewrite `frontend/src/lib/api.js` to enhance the existing `apiFetch` function. Keep the existing CSRF and credentials behavior (from Phase 21). Add:

1. **Import from constants.js**: `import { API_BASE_URL, DEFAULT_TIMEOUT_MS, MAX_RETRIES, RETRY_BACKOFF_MS } from './constants'`

2. **Import toast**: `import { toast } from '@/hooks/use-toast'` for global error display

3. **getCsrfToken()** — keep existing implementation (reads csrf_token cookie)

4. **Timeout via AbortController** (per API-01):
   - Create an AbortController with `setTimeout(() => controller.abort(), timeoutMs)` defaulting to `DEFAULT_TIMEOUT_MS` (15000ms)
   - Pass `signal: controller.signal` to fetch
   - On AbortError: show toast with title "Request timed out" and description "Please try again", then throw
   - Allow callers to override timeout via `options.timeout` (in ms) — extract before passing to fetch
   - Allow callers to pass their own `options.signal` which should be composed with the timeout signal (if caller passes signal, use it; don't add timeout abort to avoid conflicts)

5. **Retry on 5xx** (per API-02):
   - After receiving a 5xx (status >= 500), wait `RETRY_BACKOFF_MS` (1000ms) then retry once
   - Only retry GET requests and idempotent requests (do NOT retry POST/PUT/PATCH/DELETE to avoid double-submission)
   - Actually: retry ALL methods once on 5xx since the spec says "1 retry on 5xx" without method restriction. The backend should be idempotent. Implement as: if `attempt < MAX_RETRIES && res.status >= 500`, wait RETRY_BACKOFF_MS, then retry
   - After retry exhaustion, fall through to error handling

6. **Global error handling** (per API-03):
   - 401: `window.location.href = '/auth?expired=1'` then throw (keep existing behavior)
   - 403: Show toast with `{ title: "Permission denied", description: "You don't have access to this resource.", variant: "destructive" }`, then throw `new Error("Permission denied")`
   - 5xx (after retry): Show toast with `{ title: "Server error", description: "Something went wrong. Please try again later.", variant: "destructive" }`, then return the response (let caller decide)
   - 4xx (other than 401/403): Return the response as-is (caller handles validation errors, 404s, etc.)

7. **Export signature**:
   ```javascript
   export async function apiFetch(path, options = {})
   ```
   - Accepts `path` (string starting with `/api/...`)
   - Returns the raw `Response` object (backward compatible — callers still call `.json()`)
   - The `options.timeout` override is extracted before spreading to fetch

The complete file should be ~100-120 lines. Keep the JSDoc header comment updated to reflect the new capabilities.

IMPORTANT: Do NOT auto-parse JSON. Return raw Response. This maintains backward compatibility with all 100+ existing call sites that do `res.json()` or check `res.ok` before parsing.

IMPORTANT: The `toast()` function from use-toast.js is a standalone function (not a hook) — it can be called outside React components.
