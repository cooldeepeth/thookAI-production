<task type="auto">
  <name>Task 1: Run dependency audits and resolve or risk-accept findings</name>
  <files>.planning/SHIP-CHECKLIST.md</files>
  <read_first>backend/requirements.txt, frontend/package.json</read_first>
  <action>
    Step 1 — Frontend audit (SHIP-02):
    ```
    cd frontend && npm audit --audit-level=high
    ```
    - If exit code 0: proceed.
    - If exit code non-zero (high/critical vulns found):
      a. Run `npm audit fix` to apply safe auto-fixes.
      b. Re-run `npm audit --audit-level=high`.
      c. For any remaining high/critical that cannot be auto-fixed: record each as a
         risk-acceptance note in the SUMMARY with: package name, severity, CVE if known,
         why it is acceptable (e.g., dev-only dep, no attack surface in prod), and
         mitigation plan. Vulns in devDependencies that are never bundled into the
         production build (e.g., jest, react-scripts internals) are acceptable to
         risk-accept with a written note.
    Record the final npm audit output (or "exit 0 — no findings") in the SUMMARY.

    Step 2 — Backend audit (SHIP-03):
    Install pip-audit if not available: `pip install pip-audit`
    Run: `cd backend && pip-audit -r requirements.txt`
    - If exit code 0: proceed.
    - If non-zero (critical vulns): attempt `pip-audit -r requirements.txt --fix` for
      safe auto-upgrades, then update requirements.txt and re-run.
    - For unfixable critical findings: record same risk-acceptance note as Step 1.
    Record the final pip-audit output in the SUMMARY.

    Step 3 — Hardcoded secrets sweep (SHIP-06 additional check):
    Run these greps and record results:
    ```
    grep -rn "sk-ant-\|sk-proj-\|AIza\|AKIA\|whsec_\|sk_live_" backend/ --include="*.py" \
      | grep -v "__pycache__" | grep -v "tests/" | grep -v "llm_keys.py" | grep -v "config.py"
    grep -rn "sk-ant-\|sk-proj-\|AIza\|AKIA\|whsec_\|sk_live_" frontend/src/ \
      --include="*.js" --include="*.jsx" | grep -v "__tests__" | grep -v "node_modules"
    ```
    If any matches: remove the hardcoded value and replace with an environment variable
    reference (backend: `settings.*`, frontend: `process.env.REACT_APP_*`).
    If no matches: record "no hardcoded secrets found" in SUMMARY.
  </action>
  <verify>
    <automated>cd frontend && npm audit --audit-level=high; echo "npm-audit-exit:$?"</automated>
  </verify>

  </acceptance_criteria>
  <done>Both audits complete. Zero unaddressed high/critical vulnerabilities. Secret sweep clean.</done>
