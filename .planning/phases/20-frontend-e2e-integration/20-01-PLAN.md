  <action>
1. Create a root-level `package.json` (NOT inside frontend/) with:
   - `"name": "thookai-e2e"`
   - `"private": true`
   - `"devDependencies": { "@playwright/test": "^1.50.0" }`
   - `"scripts": { "test:e2e": "playwright test", "test:e2e:ui": "playwright test --ui" }`

2. Run `npm install` at repo root, then `npx playwright install chromium` (Chromium only — no Firefox/WebKit to save CI time).

3. Create `playwright.config.ts` at repo root:
   - `testDir: './e2e'`
   - `timeout: 60000` (60s per test — content generation can be slow)
   - `retries: process.env.CI ? 2 : 0`
   - `workers: process.env.CI ? 1 : undefined` (serial in CI for stability)
   - `use: { baseURL: 'http://localhost:3000', trace: 'on-first-retry', screenshot: 'only-on-failure' }`
   - `projects: [{ name: 'chromium', use: { ...devices['Desktop Chrome'] } }]`
   - `webServer: [` (array of two servers)
     - `{ command: 'cd backend && uvicorn server:app --host 0.0.0.0 --port 8001', url: 'http://localhost:8001/health', reuseExistingServer: !process.env.CI, timeout: 120000 }`
     - `{ command: 'cd frontend && npm start', url: 'http://localhost:3000', reuseExistingServer: !process.env.CI, timeout: 120000, env: { BROWSER: 'none', REACT_APP_API_URL: 'http://localhost:8001/api' } }`
   - `]`
   - `reporter: [['html', { open: 'never' }], ['list']]`

4. Create `e2e/helpers/auth.ts`:
   - `signUp(page, { email, password, name })` — navigates to `/auth`, fills signup form, submits, waits for redirect
   - `logIn(page, { email, password })` — navigates to `/auth`, fills login form, submits, waits for dashboard redirect
   - `getAuthToken(page)` — evaluates `localStorage.getItem('token')` in page context, returns string
   - Use `page.waitForURL('**/dashboard**')` (not waitForTimeout) after login

5. Create `e2e/helpers/mock-api.ts`:
   - `mockLLMResponse(page, response)` — intercepts `/api/content/generate` route via `page.route()` and returns deterministic JSON with status "reviewing" and a mock draft
   - `mockStripeCheckout(page, sessionUrl)` — intercepts `/api/billing/checkout` route and returns mock checkout session URL
   - `mockOnboardingLLM(page)` — intercepts `/api/onboarding/generate-persona` and returns a mock persona engine object
   - All mocks use `page.route()` for request interception (Playwright built-in, no external tools)

6. Create `e2e/fixtures.ts`:
   - Export a custom `test` fixture extending `@playwright/test` base
   - `authenticatedPage` fixture: creates a fresh page, calls `signUp` with a unique email (using timestamp), returns page with auth token set
   - `mockPage` fixture: creates a page with all LLM mocks pre-applied via `mock-api.ts` helpers

7. Create `e2e/smoke.spec.ts` — a trivial smoke test:
   - `test('landing page loads', async ({ page }) => { await page.goto('/'); await expect(page.locator('body')).toBeVisible(); })`
   - `test('auth page loads', async ({ page }) => { await page.goto('/auth'); await expect(page.locator('body')).toBeVisible(); })`
   - `test('health endpoint responds', async ({ request }) => { const res = await request.get('http://localhost:8001/health'); expect(res.ok()).toBeTruthy(); })`

IMPORTANT: Use `waitForSelector`, `waitForURL`, `waitForResponse` throughout — NEVER use `waitForTimeout` (research finding: causes flakiness).

  <action>
1. Check existing `.github/workflows/` directory for CI patterns (Python version, Node version, caching strategy).

2. Create `.github/workflows/e2e.yml`:
   ```yaml
   name: E2E Tests
   on:
     pull_request:
       branches: [dev, main]
     workflow_dispatch:

   jobs:
     playwright:
       runs-on: ubuntu-latest
       timeout-minutes: 30
       services:
         mongo:
           image: mongo:7.0
           ports: ["27017:27017"]
           options: --health-cmd "mongosh --eval 'db.runCommand({ping:1})'" --health-interval 10s --health-timeout 5s --health-retries 5
         redis:
           image: redis:7-alpine
           ports: ["6379:6379"]
           options: --health-cmd "redis-cli ping" --health-interval 10s --health-timeout 5s --health-retries 5
       env:
         MONGO_URL: mongodb://localhost:27017
         DB_NAME: thookai_test
         JWT_SECRET_KEY: test-jwt-secret-e2e
         FERNET_KEY: <generate a valid 32-byte base64 fernet key>
         ANTHROPIC_API_KEY: sk-test-mock
         REDIS_URL: redis://localhost:6379/0
         REACT_APP_API_URL: http://localhost:8001/api
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: '3.11'
             cache: pip
             cache-dependency-path: backend/requirements.txt
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: npm
             cache-dependency-path: |
               package-lock.json
               frontend/package-lock.json
         - name: Install backend deps
           run: pip install -r backend/requirements.txt
         - name: Install frontend deps
           run: cd frontend && npm ci
         - name: Install Playwright
           run: npm ci && npx playwright install chromium --with-deps
         - name: Run E2E tests
           run: npx playwright test --reporter=list
         - uses: actions/upload-artifact@v4
           if: always()
           with:
             name: playwright-report
             path: playwright-report/
             retention-days: 7
   ```

3. Note: Node 20 used for Playwright CI job (research finding: Playwright recommends Node 20+). The frontend CRA still builds fine under Node 20.

4. Generate a valid Fernet key for the env block: `python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` and embed the output.
  </action>
