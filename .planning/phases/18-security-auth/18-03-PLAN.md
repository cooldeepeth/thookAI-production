  <action>
    Create `backend/tests/security/test_input_validation.py` with comprehensive input validation attack tests.

    Use httpx.AsyncClient + ASGITransport for route tests. Use the `from server import app` pattern inside a `patch("database.db")` context.

    **class TestNoSQLInjection** (per SEC-05):
    - `test_login_email_nosql_injection_gt_returns_422_or_401`: POST /api/auth/login with body `{"email": {"$gt": ""}, "password": "test"}`. Pydantic EmailStr should reject this with 422 (not a valid email string). Assert status != 200.
    - `test_login_email_nosql_injection_ne_returns_422`: POST with `{"email": {"$ne": null}, "password": "test"}`. Assert 422.
    - `test_login_email_nosql_injection_regex_returns_422`: POST with `{"email": {"$regex": ".*"}, "password": "test"}`. Assert 422.
    - `test_login_email_nosql_injection_where_returns_422`: POST with `{"email": {"$where": "1==1"}, "password": "test"}`. Assert 422.
    - `test_register_name_nosql_injection_does_not_leak_data`: POST /api/auth/register with valid email, valid password, but name=`{"$gt": ""}`. Pydantic coerces name to string, so this becomes the literal string `{"$gt": ""}`. Assert register succeeds (200) but the stored name is the literal string, NOT a query operator. Patch db.users.find_one to return None (new user), capture insert_one argument, verify name field is a string.
    - `test_login_password_nosql_injection_does_not_bypass_auth`: POST with valid email but password=`{"$ne": ""}`. Pydantic accepts any string for password, but verify_password will fail since bcrypt.verify rejects dict. Assert 422 (Pydantic rejects non-string) or 401 (password comparison fails).
    - `test_content_generate_nosql_injection_in_raw_input`: POST /api/content/generate (authenticated) with raw_input=`{"$gt": ""}`. Verify it doesn't cause a MongoDB operator injection — the value should be stored as-is. Mock auth to return a valid user, mock db operations. Assert status is not 500.

    **class TestXSSPrevention** (per SEC-05):
    - `test_register_name_xss_stored_but_not_executed`: Register with name=`<script>alert(1)</script>`. The name should be stored as-is (server stores strings, frontend must escape on render). Assert register returns 200 and the name in the response is the literal XSS string (proving the API returns JSON, not rendered HTML).
    - `test_xss_in_login_email_rejected_by_email_validator`: POST /api/auth/login with email=`<script>alert(1)</script>`. EmailStr validation rejects it. Assert 422.
    - `test_xss_in_content_raw_input_stored_as_string`: POST content generation with raw_input=`"><img src=x onerror=alert(1)>`. Mock pipeline to succeed. Assert the stored raw_input is the literal string.
    - `test_json_response_content_type_prevents_xss`: Any API response has Content-Type: application/json (not text/html), which prevents browser XSS execution. Assert response content-type header.
    - `test_csp_header_blocks_inline_scripts`: (Covered by SEC-04 but reinforce here.) Response has "Content-Security-Policy: default-src 'none'" which blocks all script execution even if XSS payload were rendered.

    **class TestPathTraversal** (per SEC-05):
    - `test_upload_filename_path_traversal_blocked`: If upload endpoint exists, POST with filename=`../../../etc/passwd`. Verify the file is NOT saved outside the upload directory. Mock the upload handler. Assert the sanitized path does not contain `..`.
    - `test_obsidian_vault_path_traversal_blocked`: If obsidian service has path config, verify `../../etc/passwd` as vault path is rejected at the service level. Mock the obsidian service. Assert error or path sanitization.
    - `test_path_traversal_url_encoded_blocked`: Try `%2e%2e%2f%2e%2e%2f` variants in file paths. Assert rejected.

    **class TestRequestSizeLimits** (per SEC-05):
    - `test_oversized_request_body_returns_413`: Build a minimal FastAPI app with InputValidationMiddleware. Send a POST with Content-Length header > 10MB (10485761 bytes). Assert 413. Use TestClient (sync).
    - `test_normal_request_body_passes_validation`: POST with Content-Length 1000. Assert NOT 413.
    - `test_missing_content_length_still_processed`: POST without Content-Length header. Assert the request is still processed (not rejected).

    Total: ~20 input validation tests.

  <action>
    Create `backend/tests/security/test_owasp_top10.py` with systematic checks for each OWASP Top 10 2021 category. Each test class maps to an OWASP category. Tests that overlap with other SEC test files are intentional — this file serves as the comprehensive checklist.

    Use httpx.AsyncClient + ASGITransport and/or TestClient as appropriate.

    **class TestA01BrokenAccessControl** (OWASP A01):
    - `test_unauthenticated_request_to_protected_route_returns_401`: GET /api/auth/me without token. Assert 401.
    - `test_unauthenticated_request_to_content_returns_401`: GET /api/content (no token). Assert 401.
    - `test_unauthenticated_request_to_dashboard_returns_401`: GET /api/dashboard (no token). Assert 401.
    - `test_cors_preflight_allows_configured_origin_only`: OPTIONS request. Verify Access-Control-Allow-Origin is restricted (not "*" in production). Check server.py CORS config.

    **class TestA02CryptographicFailures** (OWASP A02):
    - `test_password_stored_as_bcrypt_hash_not_plaintext`: Register user, capture insert_one arg. Assert hashed_password starts with "$2b$" (bcrypt prefix), NOT equal to plaintext.
    - `test_jwt_uses_hs256_not_none`: Create a token via create_access_token(). Decode header without verification. Assert alg == "HS256".
    - `test_auth_cookie_httponly_flag_set`: Register and check response Set-Cookie header contains "httponly". Parse cookie attributes.
    - `test_oauth_tokens_encrypted_before_storage`: (Reinforce SEC-02) Verify _encrypt_token output != input.

    **class TestA03Injection** (OWASP A03):
    - `test_nosql_injection_in_email_blocked_by_pydantic`: POST /api/auth/login with email={"$gt":""}. Assert 422.
    - `test_nosql_injection_in_password_blocked`: POST with password={"$ne":""}. Assert 422 or 401 (Pydantic may coerce to string).
    - `test_mongo_queries_use_parameterized_not_string_concat`: (Static assertion) Read auth.py source, verify no f-string or .format() in db.find_one() calls. Use grep or direct assertion on source code.

    **class TestA04InsecureDesign** (OWASP A04):
    - `test_password_policy_rejects_weak_passwords`: Import PasswordPolicy. Assert validate("password") returns False. Assert validate("12345678") returns False. Assert validate("TestPass123!") returns True.
    - `test_password_policy_rejects_common_passwords`: Assert validate("password123") returns False, validate("qwerty") returns False.
    - `test_password_max_length_enforced`: Assert validate("A" * 129 + "1!") returns False (>128 chars).

    **class TestA05SecurityMisconfiguration** (OWASP A05):
    - `test_debug_mode_off_in_production_config`: Import settings. Assert is_production implies debug is False (or check that ENVIRONMENT != "development" doesn't leak stack traces).
    - `test_no_default_secret_in_production`: Import settings. If jwt_secret_key is empty/None and ENVIRONMENT=production, decode_token raises JWTError (verified in Phase 17 P0 fix).
    - `test_security_headers_present_on_error_responses`: Build app with SecurityHeadersMiddleware. Request a non-existent route. Assert 404 response STILL has security headers.

    **class TestA07IdentificationFailures** (OWASP A07):
    - `test_login_error_message_does_not_leak_user_existence`: POST /api/auth/login with non-existent email. Assert response message is "Invalid email or password" (same as wrong password). Do NOT reveal "email not found" vs "wrong password".
    - `test_login_error_message_same_for_google_user`: Login with email belonging to Google auth user. Assert same "Invalid email or password" message (doesn't reveal auth method).
    - `test_register_duplicate_email_returns_400_generic`: Register with existing email. Assert 400 detail is "Email already registered" (this is acceptable — email existence is revealed on register, which is standard practice).

    **class TestA08SoftwareIntegrity** (OWASP A08):
    - `test_stripe_webhook_requires_signature`: POST /api/billing/webhook without Stripe-Signature header. Assert 400 or 401 (not processed).

    **class TestA09Logging** (OWASP A09):
    - `test_failed_login_logs_warning`: Patch logger. POST invalid credentials. Assert logger.warning or logger.info was called (login failure is logged for audit trail). NOTE: If the implementation doesn't log, skip with a note.

    Total: ~20 OWASP tests.
