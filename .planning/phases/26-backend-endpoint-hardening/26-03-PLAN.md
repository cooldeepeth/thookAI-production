<objective>
Add Pydantic Field() constraints and field_validators to key POST endpoint request models in auth, content, onboarding, persona, and uploads routes.

Purpose: Currently most POST request models use bare BaseModel fields without constraints. Validation is done manually in the handler with if-checks raising HTTPException(400). Pydantic Field() constraints run before the handler and produce 422 with field-level paths — better for frontend clients and removes boilerplate from handlers.

Output: 5 route files updated with field constraints. Manual length/range checks removed where replaced by Pydantic equivalents. Existing validation logic for cross-field checks (platform × content_type) stays manual.
</objective>

```python
from pydantic import BaseModel, Field, field_validator, EmailStr
from typing import Optional, List, Literal

# String constraints
name: str = Field(min_length=1, max_length=100)
raw_input: str = Field(min_length=5, max_length=50000)

# Numeric range constraints
stability: float = Field(default=0.5, ge=0.0, le=1.0)
duration: int = Field(default=5, ge=1, le=300)

# Field validator for normalization + validation (use when Literal would break case-insensitive inputs)
@field_validator("platform")
@classmethod
def normalize_platform(cls, v: str) -> str:
    normalized = v.lower()
    allowed = {"linkedin", "x", "instagram"}
    if normalized not in allowed:
        raise ValueError(f"platform must be one of {sorted(allowed)}")
    return normalized
```
