**Step 2 — Add Stripe sk_test_ guard to stripe_service.py:**

Read `backend/services/stripe_service.py`. Find the `__init__` method or wherever `stripe.api_key` is set (look for `stripe.api_key = settings.stripe.secret_key` or similar).

Add the following guard immediately after the key is set:

```python
# PERF-09: Production guard — detect test keys in production environment
import logging as _logging
_svc_logger = logging.getLogger(__name__)
if (
    settings.app.environment == "production"
    and settings.stripe.secret_key
    and settings.stripe.secret_key.startswith("sk_test_")
):
    _svc_logger.critical(
        "LAUNCH-BLOCKER: STRIPE_SECRET_KEY starts with 'sk_test_' in production environment. "
        "Live payments will NOT be processed. Set STRIPE_SECRET_KEY to a 'sk_live_*' key in Railway."
    )
```

Use the existing `logger` if it is already defined at module level in the file — do not redeclare it if `logger = logging.getLogger(__name__)` already exists. Remove the `_svc_logger` alias in that case and use `logger.critical(...)`.

Read `backend/config.py`. In the `StripeConfig` dataclass, add a helper method if one does not already exist:

```python
def is_live_mode(self) -> bool:
    """Returns True if the Stripe key is a live key (not a test key)."""
    return bool(self.secret_key and self.secret_key.startswith("sk_live_"))
```

If `StripeConfig` already has a similar method, skip this addition.
