**Step 1 — Create `.lighthouserc.json` at the repo root:**

```json
{
  "ci": {
    "collect": {
      "numberOfRuns": 3,
      "url": [
        "${PROD_URL:-https://thook.ai}",
        "${PROD_URL:-https://thook.ai}/auth",
        "${PROD_URL:-https://thook.ai}/dashboard"
      ]
    },
    "assert": {
      "assertions": {
        "categories:performance": ["error", { "minScore": 0.9 }],
        "categories:accessibility": ["warn", { "minScore": 0.85 }]
      }
    },
    "upload": {
      "target": "filesystem",
      "outputDir": ".lighthouseci"
    }
  }
}
```

Note: The LHCI collect URL array does not expand shell env vars — the actual URL must be passed via `--url` flag on the CLI. The `.lighthouserc.json` is kept as the config reference; the run command uses explicit URLs.

**Step 2 — Run LHCI:**

Read the 35-02-SUMMARY.md to confirm the bundle optimization was deployed. Then:

Determine the production URL: check for `PROD_URL` env var. If not set, use `https://thook.ai`.

Run:
```bash
# Three pages, 3 runs each
PROD_URL="${PROD_URL:-https://thook.ai}"

npx @lhci/cli@0.15.1 collect \
  --url="$PROD_URL" \
  --url="$PROD_URL/auth" \
  --url="$PROD_URL/dashboard" \
  --numberOfRuns=3 \
  --config=.lighthouserc.json 2>&1 | tee /tmp/lhci-output.txt

npx @lhci/cli@0.15.1 assert \
  --config=.lighthouserc.json 2>&1 | tee -a /tmp/lhci-output.txt
```

**If PROD_URL is not a deployed production instance** (e.g., nothing at thook.ai yet), fall back to local build:
```bash
cd /Users/kuldeepsinhparmar/thookAI-production/frontend
npm run build 2>&1 | tail -5
npx serve -s build -p 5000 &
SERVE_PID=$!
sleep 3

npx @lhci/cli@0.15.1 collect \
  --url="http://localhost:5000" \
  --url="http://localhost:5000/auth" \
  --url="http://localhost:5000/dashboard" \
  --numberOfRuns=3 2>&1 | tee /tmp/lhci-output.txt

kill $SERVE_PID 2>/dev/null || true
```

Note in the report whether scores were measured against production URL or local build.
