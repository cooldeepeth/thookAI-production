Create `backend/routes/strategy.py` with three endpoints:

1. `GET /api/strategy` (DASH-05):
   - Auth: `current_user: dict = Depends(get_current_user)`
   - Query params: `status: str = "pending_approval"`, `limit: int = 20`
   - Query: `db.strategy_recommendations.find({"user_id": user_id, "status": status}, {"_id": 0}).sort("created_at", -1).limit(limit).to_list(limit)`
   - Serialize datetime fields (`created_at`, `dismissed_at`, `expires_at`, `approved_at`) using `.isoformat()` — same pattern as `routes/notifications.py` lines 70-71
   - Return: `{"cards": [...], "count": len(cards)}`

2. `POST /api/strategy/{recommendation_id}/approve` (DASH-05, DASH-02):
   - Auth: `current_user: dict = Depends(get_current_user)`
   - Call: `await handle_approval(user_id=current_user["user_id"], recommendation_id=recommendation_id)`
   - If result contains `"error": "not_found"`, raise `HTTPException(status_code=404, detail="Recommendation not found or already actioned")`
   - Return the result dict (contains `approved: True` and `generate_payload` with `platform`, `content_type`, `raw_input`)
   - The route does NOT call `/api/content/create` itself — frontend fires that call after receiving generate_payload

3. `POST /api/strategy/{recommendation_id}/dismiss` (DASH-05):
   - Auth: `current_user: dict = Depends(get_current_user)`
   - Optional body: `reason: Optional[str] = None` (use a Pydantic BaseModel `DismissRequest` with optional `reason` field)
   - Call: `await handle_dismissal(user_id=current_user["user_id"], recommendation_id=recommendation_id, reason=body.reason if body else None)`
   - If result contains `"error": "not_found"`, raise `HTTPException(status_code=404, detail="Recommendation not found or already actioned")`
   - Return the result dict (contains `dismissed: True`, `topic_suppressed_until`, `needs_calibration_prompt`)
