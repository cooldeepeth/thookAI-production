**Step 1 — Run npm audit:**
```bash
cd frontend && npm audit --audit-level=high --json 2>&1
```

**Step 2 — Triage each high/critical finding:**
Classify each vulnerability:
- **FIX (runtime)**: Package is in `"dependencies"` (not devDependencies), a safe npm version range update exists, and `npm audit fix` will not break the CRA build
- **ACCEPT (devDep)**: Package is in `"devDependencies"` or is a transitive dep of `react-scripts` build tooling — it runs only at build time, never in the browser bundle
- **ACCEPT (no fix)**: No non-breaking fix available (e.g., react-scripts 5.0.1 requires an exact transitive dep version that has a CVE but upgrading breaks the build)

**Key context for CRA projects:** `react-scripts` 5.0.1 bundles webpack, babel, postcss, and many build tools with CVEs that are never shipped to the browser. Forcing upgrades via `--force` can break the entire build. Accept these with documentation.

**Step 3 — Apply safe runtime fixes only:**
If any packages in `"dependencies"` (not devDependencies) have fixable CVEs, run:
```bash
npm audit fix --audit-level=high
```
THEN immediately run:
```bash
npm test -- --watchAll=false 2>&1 | tail -10
```
to confirm the test suite still passes. If tests break, revert the fix and document as ACCEPT.

Do NOT run `npm audit fix --force` — this can cause incompatible major version bumps.
