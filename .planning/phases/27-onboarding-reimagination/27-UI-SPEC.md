## Spacing Scale

Declared values (multiples of 4 — maps to Tailwind scale):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge padding |
| sm | 8px | `p-2` / `gap-2` | Inline spacing between icon and label |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, step content padding |
| xl | 32px | `p-8` / `gap-8` | Page-level content padding |
| 2xl | 48px | `py-12` | Major vertical breaks between wizard sections |
| 3xl | 64px | `py-16` | Not used in wizard — reserved for landing |

Exceptions:
- Mic button touch target: minimum 44px × 44px (accessibility — WCAG 2.5.5). Use `w-11 h-11` (44px).
- Progress stepper line connector: `w-8 h-px` (fixed 2px height, 32px width). Not on the scale — decorative only.
- Step counter badge: `w-6 h-6` (24px) — matches existing stepper pattern.

Source: existing `index.jsx` stepper, DESIGN-SYSTEM.md, WCAG 2.5.5

## Step 1: WritingStyleStep — Visual Contract

This is an extension of the existing PhaseOne. The existing two-card selection UI ("existing creator" / "starting out") is preserved. Only the post-analysis result view changes.

### Post-Analysis Result View (new fingerprint confirmation)

After `analyze-posts` API call returns, instead of a single prose analysis block:

1. Show a `card-thook p-5` container with:
   - Section heading: "Your writing fingerprint" (`font-display font-bold text-xl text-white`)
   - Prose analysis in `text-sm text-zinc-300 leading-relaxed` (existing `result.analysis` text)
   - If `result.detected_patterns` is an array: render each pattern as a tag chip: `badge-lime` class (from `index.css`)
2. Below the card, two buttons side by side:
   - Primary: "This is me" — `btn-primary` — calls `onContinue(result)` to advance to Step 2
   - Ghost: "Edit my posts" — `btn-ghost` — resets `result` state back to the textarea view

Copy:
- Confirmation heading: "Your writing fingerprint"
- Primary CTA: "This is me"
- Secondary action: "Edit my posts"
- Analysis intro label: `text-xs text-zinc-600 uppercase tracking-widest font-mono` — "Style Analysis"

## Step 2: VoiceRecordingStep — Visual Contract

Full-page centered layout matching existing step pattern (`flex items-center justify-center min-h-full p-8`).

### States

**Idle (no recording yet):**
- Step label: `text-lime text-xs font-semibold uppercase tracking-widest mb-3` — "Step 2 of 5"
- Heading: "Let's hear your voice" (`font-display font-bold text-3xl text-white`)
- Subheading: "Read any sentence aloud for 15–30 seconds. We use voice patterns to calibrate your Persona Engine." (`text-zinc-500 text-sm max-w-md mx-auto text-center`)
- Mic button: 80px × 80px circle (`w-20 h-20`), `bg-lime/10 border-2 border-lime/20 rounded-full`, centered Mic icon `text-lime` size 32. On hover: `border-lime/50 bg-lime/20`.
- Below mic: "Click to start recording" in `text-zinc-500 text-xs`
- Skip link: `text-zinc-600 text-xs underline cursor-pointer` — "Skip voice sample" — below the button row. Skipping passes `voice_sample_url: null` to generate-persona.

**Recording active:**
- Mic button changes to Stop icon (`Square` from lucide), `bg-red-500/10 border-2 border-red-400/40`, pulse ring: `animate-pulse-lime` on outer ring div (`w-24 h-24 rounded-full border border-lime/20`)
- Timer: `00:28` countdown in `font-mono text-lime text-xl font-bold` centered below the button
- Recording indicator dot: `w-2 h-2 bg-red-400 rounded-full animate-pulse` left of timer text
- Progress arc: not required — use text timer only (keeps implementation simple)

**Recorded (playback state):**
- Green checkmark icon in a `w-12 h-12 bg-lime/15 rounded-2xl` container centered above controls
- Heading: "Voice sample recorded" (`font-display font-bold text-2xl text-white`)
- Native HTML `<audio controls>` element with `src={audioUrl}` — styled with `w-full rounded-xl` and `className="audio-player"`. No custom audio player — use browser native controls.
- Two buttons:
  - Ghost: "Record again" — resets state to idle
  - Primary: "Sounds good, continue" — advances to Step 3

**Error (mic permission denied or HTTPS issue):**
- `AlertCircle` icon `text-red-400` size 24
- Error text: "Microphone access was denied. Please allow microphone permissions in your browser settings and try again." (`text-red-400 text-sm text-center max-w-sm`)
- Skip button: "Continue without voice sample" — `btn-ghost text-sm`

### Accessibility
- Mic button has `aria-label="Start voice recording"` / `aria-label="Stop recording"` depending on state.
- `aria-live="polite"` region wrapping timer for screen reader countdown announcements.
- Minimum touch target 44×44px for all buttons (mic button is 80px — satisfies).

- Grid: `grid grid-cols-2 md:grid-cols-3 gap-4` — 6 cards, 2 columns mobile, 3 columns tablet+

## Save-as-You-Go — Interaction Contract

Implemented in `OnboardingWizard` (`index.jsx`). Not a visible UI element — a behavior contract.

| Trigger | Action |
|---------|--------|
| Advance from Step 1 | `saveDraft({ step: 2, postsAnalysis, writingSamples })` |
| Advance from Step 2 | `saveDraft({ step: 3, voiceSampleUrl })` |
| Advance from Step 3 | `saveDraft({ step: 4, visualPreference })` |
| Answer each interview question | `saveDraft({ step: 4, interviewAnswers: [...] })` |
| Persona generated successfully | `clearDraft()` |
| Page reload | On mount: read draft, restore step + state if `!user.onboarding_completed` |

Draft key: `'thook_onboarding_draft_v2'` (v2 suffix prevents conflict with any prior draft).

Draft does NOT store audio blobs (they are browser-memory only). If user reloads during Step 2, they return to Step 2 with idle state (no blob) and can re-record.

## Animation Contract

All step transitions use the existing `AnimatePresence mode="wait"` with `motion.div` pattern:

```jsx
initial={{ opacity: 0, x: 20 }}
animate={{ opacity: 1, y: 0 }}
exit={{ opacity: 0, x: -20 }}
transition={{ duration: 0.3 }}
```

Within steps:
- New elements entering after an async action (fingerprint result, recorded state): `initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.4 }}`
- Palette cards stagger: `transition={{ delay: 0.05 * index }}` on each `motion.div` card
- Mic button state transitions: CSS `transition-all 0.2s ease` only (no Framer Motion — keeps it simple)
- Pulse ring on recording: `animate-pulse-lime` Tailwind animation (already defined in `tailwind.config.js`)

Source: `DESIGN-SYSTEM.md`, existing `index.jsx`, `PhaseOne.jsx`

## Accessibility Contract

- Every interactive element has an accessible label (`aria-label`, or visible text that is the label).
- Mic button: `aria-label` updates dynamically with recording state.
- Palette cards: `role="radio"` on each, `aria-checked={isSelected}`, wrapped in `role="radiogroup"`.
- Timer region: `aria-live="polite"` so screen readers announce remaining time.
- Keyboard navigation: All buttons and palette cards reachable via Tab. Palette card selection via Space/Enter.
- Focus ring: Use `.focus-ring:focus-visible` class (defined in `index.css`) on custom buttons not using shadcn `Button`.
- Color contrast: Lime (#D4FF00) on black (#050505) — contrast ratio ~14:1 (exceeds WCAG AA 4.5:1 for normal text).
