<objective>
Fix two critical publish-pipeline bugs that cause silent data loss:
1. `_publish_to_platform()` in content_tasks.py sends the raw Fernet-encrypted token string to platform APIs (garbage bytes → 401) and discards the rich result dict (post_id, post_url lost)
2. `scheduled_tasks._process_scheduled_posts()` does not write publish_results to the linked content_jobs document (analytics polling then silently finds nothing to query), and passes the wrong kwarg name `media_urls` instead of `media_assets`

Purpose: Unblock end-to-end publishing. Without this fix, every scheduled post fails at the HTTP level (encrypted token sent as Bearer header), and even if it somehow succeeded, metrics polling 24h later would have no post_id to query.

Output: Patched content_tasks.py, patched scheduled_tasks.py, and new pytest test cases proving the fixes work.
</objective>
