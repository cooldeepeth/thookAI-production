<objective>
Add media attachment support to the LinkedIn and X publishers, and verify the existing Instagram media path.

RESEARCH.md states:
- LinkedIn: "text-only (MVP note in code)" — no media upload path exists
- X: "No media upload path exists for X — text only"
- Instagram: Full Media Container flow already exists (uses image_url from R2)

PUBL-01 says "publish UGC posts with media" and PUBL-02 says "publish tweets/threads with media". These requirements cannot be satisfied without wiring media upload into the LinkedIn and X publishers.

**LinkedIn media flow** (UGC API):
1. POST `https://api.linkedin.com/v2/assets?action=registerUpload` to get an upload URL
2. PUT the image bytes to the upload URL
3. Include `media[0].media` (the asset URN) in the UGC post body

**X media flow** (v1.1 upload + v2 tweet):
1. POST `https://upload.twitter.com/1.1/media/upload.json` with the image (multipart or base64)
2. Attach returned `media_id_string` in the v2 tweet body as `"media": {"media_ids": [media_id]}`

**Instagram**: No change needed — the existing Media Container flow already reads `image_url` from `media_assets[0]`. This task verifies that path is reachable from the `media_assets` argument.

**Scope constraint**: Only image attachment from `image_url` (public R2 URL). Video upload is out of scope for this plan — the image bytes are fetched via httpx from the public R2 URL and uploaded to the platform.

Purpose: Satisfy PUBL-01 and PUBL-02 media requirements without replacing the existing text-only paths (text-only publishing must still work when `media_assets` is None or empty).

Output: Updated publisher.py with LinkedIn and X image upload paths, new tests proving both media flows, and VALIDATION.md populated.
