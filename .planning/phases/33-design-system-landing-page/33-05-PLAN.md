<task type="auto">
  <name>Task 2: Create og-image.png (real PNG, never SVG bytes)</name>
  <files>frontend/public/og-image.png</files>
  <action>
Create a static 1200x630 OG image at `frontend/public/og-image.png`. The output MUST be a
real PNG binary — Twitter/X, LinkedIn, and Facebook crawlers reject SVG for `og:image`.

**Step 1 — Write the SVG source.** Create `scripts/generate-og-image.js` that writes a
1200x630 SVG to `frontend/public/og-image.svg`. The script uses only Node.js built-ins
(no new npm packages). Design tokens — DO NOT change these:
- Background rect fill: `#050505`
- Top accent bar: `<rect width="1200" height="6" fill="#D4FF00"/>`
- Center title text: `fill="white" font-family="Arial" font-weight="bold" font-size="80"`,
  text content `ThookAI`, x=600 y=280 text-anchor=middle
- Tagline: `fill="#A1A1AA" font-family="Arial" font-size="32"`,
  text content `Your AI Creative Agency`, x=600 y=360
- Platforms line: `fill="#D4FF00" font-family="Arial" font-size="24"`,
  text content `LinkedIn · X · Instagram`, x=600 y=440
- Bottom-right wordmark: `fill="#D4FF00" font-family="Arial" font-size="20"`,
  text content `thook.ai`, x=1160 y=610 text-anchor=end

**Step 2 — Convert SVG to PNG.** Try the converters in this exact order. The first one
that succeeds wins. The script must call only ONE — do not chain them.

```bash
# 1. macOS built-in (always available on macOS — no install needed):
sips -s format png frontend/public/og-image.svg --out frontend/public/og-image.png 2>&1

# 2. ImageMagick (if installed):
convert frontend/public/og-image.svg frontend/public/og-image.png 2>&1

# 3. rsvg-convert (Linux/Homebrew):
rsvg-convert -w 1200 -h 630 frontend/public/og-image.svg -o frontend/public/og-image.png 2>&1
```

**Step 3 — Validate the output is a real PNG.** This is the gate that prevents the
"SVG bytes saved as .png" failure mode.

```bash
file frontend/public/og-image.png
# MUST output something containing "PNG image data" — anything else is a FAIL.
```

If `file` shows anything other than `PNG image data`, delete the broken `og-image.png`
and STOP — report the failure to the orchestrator. Do NOT proceed by copying the SVG.
Do NOT install a new npm package without first asking for approval (this would violate
CLAUDE.md rule 5).

**Step 4 — Clean up.** Delete `frontend/public/og-image.svg` after successful PNG creation
(it was a build artifact, not the deploy artifact). Delete `scripts/generate-og-image.js`
unless it would be useful to commit as a regenerator (judgment call — committing it is
fine, just make sure it's pure-Node and uses no new packages).

**Constraints:**
- No new npm packages (no `canvas`, no `sharp`, no `jimp`, no `puppeteer`).
- The deployed file at `frontend/public/og-image.png` must be a real PNG binary verified by `file(1)`.
- 1200x630 dimensions are mandatory (the verify gate doesn't strictly check dimensions, but social-card crawlers do).
- If on a non-macOS environment without ImageMagick or rsvg-convert, the executor must
  STOP and surface the failure rather than commit a fake PNG. The orchestrator can then
  install ImageMagick (`brew install imagemagick` / `apt-get install imagemagick`) and retry.
