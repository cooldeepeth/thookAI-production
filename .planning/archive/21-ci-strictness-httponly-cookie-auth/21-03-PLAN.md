<objective>
Migrate the frontend auth flow from localStorage JWT to cookie-based sessions. AuthContext, AuthPage, and the apiFetch wrapper stop reading/writing localStorage for the thook_token. All authenticated requests use credentials: 'include' and send the CSRF token header.

Purpose: AUTH-03 requires frontend reads from cookie session, not localStorage. AUTH-04 requires removal of localStorage.setItem for JWT. Note: the ~40 localStorage.getItem calls across Dashboard components will be removed in Phase 22 when apiFetch replaces all raw fetch calls. This plan focuses on the core auth entry points (AuthContext, AuthPage, api.js).
Output: Updated AuthContext.jsx, AuthPage.jsx, and api.js with zero localStorage JWT usage in auth flow.
</objective>

    Rewrite `AuthContext.jsx` to eliminate all localStorage usage for auth tokens. The cookie is set by the backend and sent automatically by the browser via `credentials: 'include'`.

    **Changes to `checkAuth` callback:**
    - Remove: `const token = localStorage.getItem("thook_token");`
    - Remove: `if (token) { headers.Authorization = \`Bearer ${token}\`; }`
    - The fetch to `/api/auth/me` should only use `credentials: "include"` — the browser sends the session_token cookie automatically.
    - After successful /api/auth/me response, also fetch the CSRF token: `fetch(\`${BACKEND_URL}/api/auth/csrf-token\`, { credentials: "include" })` and store the csrf_token in a ref or state for use by api.js. Store it in a module-level variable: `window.__CSRF_TOKEN = data.csrf_token;` (simple, accessible by api.js without prop drilling). Alternatively, read it from the cookie: `document.cookie.split('; ').find(c => c.startsWith('csrf_token='))?.split('=')[1]`. Use the cookie-reading approach since the CSRF cookie is set by the backend on every auth response.

    **Changes to Google OAuth token handling (useEffect):**
    The current flow: if `?token=` URL param exists, it saves to localStorage and calls /api/auth/me with Bearer header.
    New flow:
    - If `?token=` URL param exists, call a new one-time endpoint to exchange it: `fetch(\`${BACKEND_URL}/api/auth/me\`, { headers: { Authorization: \`Bearer ${token}\` }, credentials: "include" })`. This call will return the user AND the backend will set the session_token cookie on the response (because get_current_user validates the token and the route can set a cookie).
    - HOWEVER, the /api/auth/me GET endpoint does NOT currently set a cookie. The Google OAuth flow already sets a session_token cookie in the Google callback handler. So the token URL param is redundant with the cookie — test if removing the Bearer header still works (it should, because the Google OAuth callback already sets the cookie before redirecting to frontend).
    - Simplest fix: Remove the entire `?token=` localStorage flow. After Google OAuth callback, the cookie is already set. Just call `checkAuth()` which hits /api/auth/me with credentials:include (cookie sent automatically).
    - But keep backward compat: if `?token=` param exists, still call /api/auth/me with the Bearer header (one-time) to validate, but do NOT save to localStorage. After success, call `window.history.replaceState({}, "", "/dashboard")` and set user state. The cookie should already be set by the Google callback.
    - Remove: `localStorage.setItem("thook_token", token);`
    - Remove: `localStorage.removeItem("thook_token");` from error handler

    **Changes to `login` function:**
    - Keep as is: `const login = (userData) => setUser(userData);` — no localStorage involved.

    **Changes to `logout` function:**
    - Remove: `localStorage.removeItem("thook_token");`
    - Keep: the fetch to `/api/auth/logout` with `credentials: "include"` — this clears the cookies on the backend.
    - After logout, also clear `window.__CSRF_TOKEN` if using that approach, or do nothing since cookies are cleared.

    **Final file should have ZERO references to `localStorage`.**

    2. Add CSRF token header for state-changing requests. Read the csrf_token cookie value:
       ```javascript
       function getCsrfToken() {
         const match = document.cookie.match(/(?:^|; )csrf_token=([^;]*)/);
         return match ? decodeURIComponent(match[1]) : null;
       }
       ```
    3. In apiFetch, for non-GET requests, add the CSRF header:
       ```javascript
       const method = (options.method || 'GET').toUpperCase();
       if (method !== 'GET' && method !== 'HEAD') {
         const csrfToken = getCsrfToken();
         if (csrfToken) {
           headers['X-CSRF-Token'] = csrfToken;
         }
       }
       ```
    4. Keep `credentials: 'include'` — this sends the session_token cookie automatically.
    5. In the 401 handler, remove `localStorage.removeItem('thook_token');`. Just redirect to `/auth?expired=1`.
