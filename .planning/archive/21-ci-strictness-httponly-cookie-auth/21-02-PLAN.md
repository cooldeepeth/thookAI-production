    **Step 1: Create `backend/middleware/csrf.py`**

    Implement `CSRFMiddleware` as a Starlette BaseHTTPMiddleware:

    ```
    class CSRFMiddleware(BaseHTTPMiddleware):
    ```

    Double-submit cookie pattern logic:
    1. Extract method from request. If method is in SAFE_METHODS (`GET`, `HEAD`, `OPTIONS`), skip CSRF check entirely.
    2. Check if the request path is in EXEMPT_PATHS (a set of paths that don't require CSRF). Exempt paths:
       - `/api/auth/login`
       - `/api/auth/register`
       - `/api/auth/google`
       - `/api/auth/google/callback`
       - `/api/auth/forgot-password`
       - `/api/billing/webhook` (Stripe webhook uses signature verification)
       - `/api/n8n/` prefix (n8n bridge uses HMAC verification)
       - `/health`
    3. Check if the request uses cookie-based auth: look for `session_token` in request.cookies. If no session_token cookie is present, the request is using header-based auth — skip CSRF check.
    4. If cookie-based auth detected: read `X-CSRF-Token` header from request. Read `csrf_token` cookie from request. If header is missing, return 403 `{"detail": "CSRF token missing"}`. If header value does not match cookie value, return 403 `{"detail": "CSRF token invalid"}`.
    5. If all checks pass, call `await call_next(request)`.

    Use `from starlette.responses import JSONResponse` for 403 responses.

    **Step 2: Update `backend/routes/auth.py`**

    Add a `set_csrf_cookie(response, csrf_value)` helper function:
    ```python
    import secrets

    def set_csrf_cookie(response: Response, csrf_value: str):
        is_dev = settings.app.is_development
        expire_days = settings.security.jwt_expire_days
        response.set_cookie(
            key="csrf_token", value=csrf_value,
            httponly=False,  # JS must read this
            secure=not is_dev,
            samesite="lax" if is_dev else "none",
            max_age=expire_days * 24 * 3600, path="/"
        )
    ```

    Update `register()` endpoint: After `set_auth_cookie(response, token)`, generate `csrf_value = secrets.token_urlsafe(32)` and call `set_csrf_cookie(response, csrf_value)`. Include `csrf_token` in the JSON response body too (for the frontend to use immediately without reading the cookie).

    Update `login()` endpoint: Same — after `set_auth_cookie(response, token)`, generate CSRF value and call `set_csrf_cookie(response, csrf_value)`. Include `csrf_token` in JSON response.

    Update `logout()` endpoint: After `response.delete_cookie(key="session_token", ...)`, also `response.delete_cookie(key="csrf_token", path="/", samesite="none" if not is_dev else "lax", secure=not is_dev)`.

    Add a new `GET /api/auth/csrf-token` endpoint that returns a fresh CSRF token and sets the csrf_token cookie. This is for page reloads where the frontend needs a fresh token. It should require authentication (Depends(get_current_user)):

    ```python
    @router.get("/csrf-token")

        set_csrf_cookie(response, csrf_value)
        return {"csrf_token": csrf_value}
    ```
