    - `RateLimiter` class with `_is_rate_limited_redis()` (Redis sliding window) and `_is_rate_limited_memory()` (in-memory fallback)
    - `is_rate_limited()` tries Redis first, falls back to in-memory on any error
    - `RateLimitMiddleware` uses the global `_rate_limiter` instance

    Verify this works by:
    1. Confirming `get_redis()` is imported from `middleware.redis_client`
    2. Confirming the fallback path: if `get_redis()` returns None or raises, in-memory is used
    3. Confirming rate limit headers are set on responses: `X-RateLimit-Limit` and `X-RateLimit-Remaining`
    4. Confirming 429 response includes `Retry-After` header

    The implementation looks correct as-is. Write a brief test in `backend/tests/test_rate_limit.py` to verify the in-memory fallback works:

    ```python
    """Tests for rate limiting middleware in-memory fallback."""
    import pytest
    from middleware.security import RateLimiter


    @pytest.mark.asyncio
    async def test_rate_limiter_in_memory_allows_under_limit():
        limiter = RateLimiter()
        is_limited, remaining = limiter._is_rate_limited_memory("test:key", limit=5)
        assert is_limited is False
        assert remaining == 4


    @pytest.mark.asyncio
    async def test_rate_limiter_in_memory_blocks_over_limit():
        limiter = RateLimiter()
        for i in range(5):
            limiter._is_rate_limited_memory("test:block", limit=5)
        is_limited, remaining = limiter._is_rate_limited_memory("test:block", limit=5)
        assert is_limited is True
        assert remaining == 0


    @pytest.mark.asyncio
    async def test_rate_limiter_async_fallback_without_redis():
        """When Redis is unavailable, is_rate_limited falls back to in-memory."""
        limiter = RateLimiter()
        is_limited, remaining = await limiter.is_rate_limited("test:async", limit=10)
        assert is_limited is False
        assert remaining >= 0
    ```
