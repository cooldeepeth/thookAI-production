<objective>
Build the Remotion render service from scratch as a standalone Express API in TypeScript.

Purpose: The Remotion service is the video/image composition engine for ThookAI. It receives render requests from the Python backend, renders compositions using Remotion's SSR API, uploads output to R2, and reports status. This is a sidecar service — no Python imports, integrated over HTTP.

Output: A complete Node.js Express service in `remotion-service/` with 5 registered compositions, bundle caching, async job queue, R2 upload, and docker-compose integration.
</objective>

       - devDependencies: typescript, @types/express, @types/node, @types/uuid, ts-node

    2. Create `remotion-service/tsconfig.json` with:
       - target: "ES2020", module: "commonjs", outDir: "./dist", rootDir: ".", strict: true
       - jsx: "react-jsx", jsxImportSource: "react"
       - include: ["server.ts", "src/**/*", "lib/**/*"]

    3. Create `remotion-service/lib/job-store.ts`:
       - Export a `Map<string, JobStatus>` where JobStatus = { status: "queued" | "rendering" | "done" | "failed"; url?: string; error?: string; compositionId: string; createdAt: number }
       - Export functions: `createJob(compositionId: string): string` (returns render_id), `getJob(id: string): JobStatus | undefined`, `updateJob(id: string, update: Partial<JobStatus>): void`
       - Use uuid v4 for render_id generation

    4. Create `remotion-service/lib/r2-upload.ts`:
       - Use `@aws-sdk/client-s3` with `S3Client` and `PutObjectCommand`
       - Configure endpoint from `R2_ENDPOINT` env var (format: `https://{account_id}.r2.cloudflarestorage.com`)
       - Configure credentials from `R2_ACCESS_KEY_ID` and `R2_SECRET_ACCESS_KEY` env vars
       - Bucket from `R2_BUCKET_NAME` env var
       - Export `async function uploadToR2(buffer: Buffer, key: string, contentType: string): Promise<string>` — returns the public URL from `R2_PUBLIC_URL` env var + "/" + key
       - If R2 env vars are missing, log warning and return a local file path (dev fallback)

    5. Create `remotion-service/server.ts`:
       - Import express, path, uuid
       - Import `bundle, selectComposition, renderMedia, renderStill` from `@remotion/renderer`
       - Import job-store and r2-upload
       - Module-level `let bundlePath: string | null = null`
       - `async function getBundlePath(): Promise<string>` — calls `bundle({ entryPoint: path.resolve("./src/index.ts") })` on first call, caches result
       - `POST /render` endpoint:
         - Accept JSON body: `{ composition_id: string, input_props: Record<string, unknown>, render_type?: "still" | "video" }`
         - Default `render_type` to "still" if composition_id is "StaticImageCard", "Infographic"; otherwise "video"
         - Create job via jobStore.createJob(composition_id)
         - Return `{ render_id }` immediately with 200
         - In background (non-blocking): call `runRender(render_id, composition_id, input_props, render_type)`
       - `async function runRender(renderId, compositionId, inputProps, renderType)`:
         - Update job status to "rendering"
         - Get bundle path via getBundlePath()
         - Call `selectComposition({ serveUrl: bundlePath, id: compositionId, inputProps })`
         - If renderType === "still": call `renderStill({ composition, serveUrl: bundlePath, output: tmpPath, inputProps })` with imageFormat "png"
         - If renderType === "video": call `renderMedia({ composition, serveUrl: bundlePath, codec: "h264", outputLocation: tmpPath, inputProps, timeoutInMilliseconds: 120000, licenseKey: process.env.REMOTION_LICENSE_KEY })`
         - Read output file as Buffer
         - Upload to R2 via `uploadToR2(buffer, "media/renders/{render_id}.{ext}", contentType)`
         - Update job status to "done" with url
         - On error: update job status to "failed" with error message
       - `GET /render/:id/status` endpoint:
         - Lookup job in jobStore
         - If not found: return 404 `{ error: "not_found" }`
         - Return job status object
       - `GET /health` endpoint: return `{ status: "ok", bundled: !!bundlePath }`
       - Listen on `process.env.PORT || 3001`
       - On startup: log "Remotion service starting on port {port}" and trigger getBundlePath() to pre-warm

    6. Update `remotion-service/Procfile` to: `web: node dist/server.js`
