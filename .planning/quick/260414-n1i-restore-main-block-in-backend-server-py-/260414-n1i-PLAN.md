**Root cause (confirmed by orchestrator):**
Commit 02697c3 (Apr 13) titled "remove os.environ.get from server.py __main__ block" deleted the entire `__main__` guard and left this comment at EOF:

```python
# Server is started via Procfile: uvicorn server:app --host 0.0.0.0 --port $PORT
# No __main__ block needed — see backend/Procfile
```

That comment is wrong. The Procfile says `web: python server.py` — not `uvicorn server:app`. Without a `__main__` block the module imports cleanly then exits 0, causing Railway crash-loops.

**Pre-regression working state (git show 4e9a083):**
```python
if __name__ == "__main__":
    import uvicorn
    port = int(os.environ.get("PORT", 8000))
    uvicorn.run("server:app", host="0.0.0.0", port=port)
```

`os` is already imported at the top of server.py. The `os.environ.get()` rule in CLAUDE.md applies to route/agent/service files — an entry-point `__main__` guard is explicitly exempt from that ban.
</context>
