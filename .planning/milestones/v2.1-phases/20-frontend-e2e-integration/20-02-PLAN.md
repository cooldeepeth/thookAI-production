   **Class `ThookAIUser(HttpUser)`:**
   - `wait_time = between(1, 3)`

   - Log final credit state.
   - Assert no negative balances detected during the run.

   **Mark all tests with `@pytest.mark.integration`** — these require Docker and are skipped in regular `pytest` runs.

   Add skip condition at module level:
   ```python
   import shutil
   import subprocess
   import time
   import pytest
   import urllib.request
   import json

   pytestmark = pytest.mark.skipif(
       not shutil.which("docker"),
       reason="Docker not available"
   )
   ```

   **Service health check config** (dict mapping service name to check info):
   ```python
   SERVICES = {
       "backend": {"url": "http://localhost:8001/health", "timeout": 60},
       "frontend": {"url": "http://localhost:3000", "timeout": 120},
       "mongo": {"port": 27017, "timeout": 30},
       "redis": {"port": 6379, "timeout": 30},
       "n8n": {"url": "http://localhost:5678/healthz", "timeout": 90},
       "lightrag": {"url": "http://localhost:9621/health", "timeout": 90},
       "remotion": {"url": "http://localhost:3001/health", "timeout": 90},
   }
   ```

   **Helper `_wait_for_url(url, timeout_sec)`:**
   - Poll every 5 seconds with `urllib.request.urlopen(url, timeout=5)`
   - Return True if 200 within timeout, False otherwise
   - Catch `URLError`, `TimeoutError`, `ConnectionRefusedError` silently during polling

   **Helper `_wait_for_port(port, timeout_sec)`:**
   - Poll every 5 seconds with socket connect to `localhost:port`
   - Return True if connection succeeds within timeout, False otherwise

   **Class `TestDockerComposeSmokeE2E06`:**

   `@pytest.fixture(scope="class")` `def compose_up(self)`:
   - Run `docker compose -f docker-compose.yml up -d --build` from repo root (use `subprocess.run` with `cwd` set to project root two levels up from tests/)
   - Yield (tests run)
   - Teardown: `docker compose -f docker-compose.yml down -v --remove-orphans`

   `def test_all_services_healthy_within_120s(self, compose_up)`:
   - Start timer
   - For each service in SERVICES, call `_wait_for_url` or `_wait_for_port`
   - Assert all services are healthy
   - Assert total elapsed time < 120 seconds
   - Print a summary table of service name + status + time-to-healthy

   `def test_backend_health_returns_ok(self, compose_up)`:
   - `GET http://localhost:8001/health` returns 200
   - Response JSON contains expected fields

   `def test_cross_service_connectivity(self, compose_up)`:
   - Backend can query MongoDB: `GET http://localhost:8001/health` returns 200 (health endpoint queries DB)
   - Backend can reach Redis: verify via health endpoint or a dedicated test endpoint
   - Document that n8n -> backend and lightrag -> mongo connectivity is verified by their respective health checks passing
