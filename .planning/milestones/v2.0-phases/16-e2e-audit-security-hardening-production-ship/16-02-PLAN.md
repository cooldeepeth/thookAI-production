  <files>docker-compose.prod.yml, docker-compose.yml</files>

  <action>
    Create `docker-compose.prod.yml` as a production override file (used via `docker compose -f docker-compose.yml -f docker-compose.prod.yml up`).

    The override file must:

    1. **Remove n8n port exposure**: Override the n8n service to NOT publish port 5678 to the host. Use `expose: ["5678"]` instead of `ports: ["5678:5678"]` so n8n is reachable only within the Docker network (backend can still call it via `http://n8n:5678`).

    2. **Add n8n basic auth**: Add environment variables `N8N_BASIC_AUTH_ACTIVE=true`, `N8N_BASIC_AUTH_USER=${N8N_BASIC_AUTH_USER}`, `N8N_BASIC_AUTH_PASSWORD=${N8N_BASIC_AUTH_PASSWORD}` to the n8n service. These are read from .env — never hardcoded.

    3. **Confirm security env vars carry through**: `N8N_BLOCK_ENV_ACCESS_IN_NODE=true` is already in docker-compose.yml base — verify the prod override does NOT accidentally remove it.

    4. **Add Docker network isolation**: Define a `thookai-internal` network. Attach backend, n8n, n8n-worker, mongo, redis, postgres-n8n, lightrag, remotion, and celery-worker to this internal network. Only backend and frontend get the `thookai-public` network (exposed to host). n8n gets ONLY the internal network.

    5. **Remotion port restriction**: In production, remotion should also use `expose: ["3001"]` instead of `ports: ["3001:3001"]` — only backend needs to reach it.

    6. **LightRAG port restriction**: In production, lightrag should use `expose: ["9621"]` instead of `ports: ["9621:9621"]`.

    Also update the base `docker-compose.yml` with a comment on the n8n ports line:
    ```yaml
    ports:
      - "5678:5678"  # DEV ONLY — overridden in docker-compose.prod.yml
    ```

    Production docker-compose.prod.yml structure:
    ```yaml
    version: "3.9"

    services:
      n8n:
        ports: !override []
        expose:
          - "5678"
        environment:
          - N8N_BASIC_AUTH_ACTIVE=true
          - N8N_BASIC_AUTH_USER=${N8N_BASIC_AUTH_USER}
          - N8N_BASIC_AUTH_PASSWORD=${N8N_BASIC_AUTH_PASSWORD}
        networks:
          - thookai-internal

      n8n-worker:
        networks:
          - thookai-internal

      lightrag:
        ports: !override []
        expose:
          - "9621"
        networks:
          - thookai-internal

      remotion:
        ports: !override []
        expose:
          - "3001"
        networks:
          - thookai-internal

      backend:
        networks:
          - thookai-internal
          - thookai-public

      frontend:
        networks:
          - thookai-public

      mongo:
        networks:
          - thookai-internal

      redis:
        networks:
          - thookai-internal

      postgres-n8n:
        networks:
          - thookai-internal

      celery-worker:
        networks:
          - thookai-internal

    networks:
      thookai-internal:
        driver: bridge
        internal: true
      thookai-public:
        driver: bridge
    ```

    NOTE: Docker Compose override files merge by default. To REMOVE ports, the prod file should set `ports: []` which replaces the base list. Test this works by running `docker compose -f docker-compose.yml -f docker-compose.prod.yml config` to see merged output.
  </action>
