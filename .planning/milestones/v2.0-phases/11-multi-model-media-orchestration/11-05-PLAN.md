<objective>
Extend the Designer agent with automated format selection (picking the best media_type for a given platform and content angle) and extend the QC agent with media-specific validation (brand consistency, dimension compliance, anti-AI-slop detection).

Purpose: The Designer agent's format selection removes the need for users to manually choose between static_image, carousel, meme, etc. — the AI picks the best format based on platform norms and content characteristics. The QC agent's media validation ensures every rendered output meets platform specs and brand standards before delivery.

Output: `select_media_format()` in designer.py, `validate_media_output()` in qc.py, and comprehensive unit tests for both.
</objective>

       Implement the function:
       ```python
       async def select_media_format(
           platform: str,
           content_text: str,
           content_angle: str = "thought_leadership",
           persona_card: Optional[Dict[str, Any]] = None,
           has_data_points: bool = False,
           has_avatar: bool = False,
       ) -> Dict[str, Any]:
           """Select the optimal media format for a given platform and content angle.

           Uses platform format preferences, content angle mapping, and available
           capabilities (data points, avatar) to pick the best media_type.

           Returns:
               {
                   "media_type": str,
                   "reason": str,  # Human-readable explanation of why this format was chosen
                   "alternatives": List[str],  # Other viable formats
                   "confidence": float,  # 0-1 confidence score
               }
           """
       ```

       Selection logic (NO LLM call needed — deterministic):
       - Get platform preferences from PLATFORM_FORMAT_PREFERENCES (default to linkedin if unknown)
       - Get angle candidates from CONTENT_ANGLE_FORMAT_MAP (default to ["static_image"])
       - Score each candidate: +3 if in platform.preferred, +1 if in platform.good, -2 if in platform.avoid
       - Bonus: +2 for "infographic" if has_data_points, +2 for "talking_head" if has_avatar
       - Bonus: +1 for "short_form_video" if content_text length > 500 (long content benefits from video)
       - Select highest-scoring format
       - Build reason string: f"Selected {media_type} for {platform}: {reason_detail}"
       - Return top pick + top 2 alternatives + confidence score (normalized 0-1)

       ```python
       async def validate_media_output(
           media_url: str,
           media_type: str,
           platform: str,
           brand_color: str = "#2563EB",
           persona_card: Optional[Dict[str, Any]] = None,
       ) -> Dict[str, Any]:
           """Validate rendered media output for brand consistency and platform compliance.

           Checks:
           1. Platform dimension compliance (image width/height within tolerance)
           2. Brand color presence (checks if brand_color hex is used)
           3. Anti-AI-slop detection (via LLM vision if available, else basic heuristics)
           4. File format correctness (PNG for stills, MP4 for video)

           Returns:
               {
                   "overall_pass": bool,
                   "checks": [
                       {"name": "platform_dimensions", "pass": bool, "detail": str},
                       {"name": "brand_consistency", "pass": bool, "detail": str},
                       {"name": "anti_slop", "pass": bool, "detail": str},
                       {"name": "file_format", "pass": bool, "detail": str},
                   ],
                   "feedback": List[str],
               }
           """
       ```

       Implementation:
       - Check 1 (platform_dimensions): Determine if media_type is image or video. Look up specs from PLATFORM_MEDIA_SPECS. For images: validate width/height within tolerance. For video: validate duration. Since we may not be able to check actual dimensions without downloading, this check verifies that the correct composition was used based on media_type and platform.
       - Check 2 (brand_consistency): Verify that brand_color was passed to the Remotion composition. This is a metadata check — the actual pixel-level check would require vision API.
       - Check 3 (anti_slop): If LLM with vision is available (anthropic API key set), use Claude to analyze the image for AI-typical artifacts (weird hands, text spelling errors, unnatural lighting). If vision not available, pass by default with note "Vision analysis unavailable."
       - Check 4 (file_format): Check URL extension — .png for stills, .mp4 for video.
       - overall_pass = all checks pass
       - Build feedback list from failed checks
