<objective>
Build the MediaOrchestrator service foundation: the MediaBrief dataclass, pipeline credit ledger, R2 pre-staging of external assets, and the core orchestrate() function that decomposes a brief into provider tasks, tracks credits per-stage, stages assets to R2, and calls the Remotion service.

Purpose: This is the Python-side orchestration layer that coordinates all media generation. Every media generation request flows through this service — it ensures credit accounting, asset staging, and Remotion integration happen correctly regardless of media type.

Output: `media_orchestrator.py` with full orchestration skeleton (media-type-specific handlers added in Plans 03-04), `RemotionConfig` in config.py, credit ledger collection with indexes, orchestrate Celery task, and comprehensive unit tests.
