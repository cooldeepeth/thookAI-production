<objective>
Wire publish results back to content_jobs and prepare analytics polling infrastructure

Purpose: When a scheduled post is published via n8n, the publisher's return value (containing post_id/tweet_id needed for platform API metrics fetching) must be written to `content_jobs.publish_results[platform]`. Without this, `social_analytics.update_post_performance()` cannot find the post identifier to call LinkedIn/X/Instagram APIs. This plan also adds the analytics due-at timestamp fields, N8nConfig workflow fields, DB indexes, and workflow map entries that Plan 02 depends on.

Output: Modified `execute_process_scheduled_posts` with publish_results write-back + analytics scheduling fields, N8nConfig with 2 new workflow fields, 2 sparse compound indexes for efficient polling queries, .env.example updated.
</objective>
