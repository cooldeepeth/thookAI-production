**STRATEGIST_SYSTEM_PROMPT (module-level constant string):**
Use the exact prompt from the research doc (Pattern 8). The prompt instructs Claude to return a JSON array of recommendation objects with keys: topic, hook_options, platform, why_now, signal_source, generate_payload. Emphasize in the prompt: "Each recommendation MUST include a clear 'why_now' rationale starting with 'Why now:'" (STRAT-03). Also emphasize: "Return between 1 and 3 recommendations" and "Return ONLY valid JSON — no markdown, no preamble".

**Function: `async def _get_eligible_users() -> List[str]`**
- Query `db.users` for users with `onboarding_completed: True`
- For each user, check `db.persona_engines` for `learning_signals.approved_count >= MIN_APPROVED_CONTENT`
- Return list of user_ids that meet the threshold
- If the persona_engine does not have `learning_signals.approved_count`, skip that user (not enough history)

**Function: `async def _get_cadence_state(user_id: str) -> dict`**
- `db.strategist_state.find_one({"user_id": user_id})` — return the document or empty dict
- Check if `cards_today_date` matches today's date string. If not, this is a new day — the card count should be treated as 0.

**Function: `async def _atomic_claim_card_slot(user_id: str) -> bool`**
- Atomic cap guard using `find_one_and_update` (from research Pitfall 2):
  ```python
  today_str = datetime.now(timezone.utc).strftime("%Y-%m-%d")
  result = await db.strategist_state.find_one_and_update(
      {"user_id": user_id, "cards_today_date": today_str, "cards_today_count": {"$lt": settings.strategist.max_cards_per_day}},
      {"$inc": {"cards_today_count": 1}},
      return_document=True,
  )
  if result is None:
      # Either no state doc for today or cap reached — try upsert for new day
      result = await db.strategist_state.find_one_and_update(
          {"user_id": user_id, "cards_today_date": {"$ne": today_str}},
          {"$set": {"cards_today_date": today_str, "cards_today_count": 1}, "$setOnInsert": {"consecutive_dismissals": 0, "halved_rate": False, "needs_calibration_prompt": False, "suppressed_topics": []}},
          upsert=True,
          return_document=True,
      )
      if result is None:
          return False  # Cap already reached
  return True
  ```
  This ensures no race condition on cadence check (STRAT-04).

**Function: `async def _is_topic_suppressed(user_id: str, topic: str) -> bool`**
- Query `db.strategy_recommendations` for a document with:
  - `user_id` matches
  - `topic` matches (case-insensitive: normalize both to lowercase before comparison)
  - `status: "dismissed"`
  - `dismissed_at >= (now - suppression_days)`
- Return True if found (STRAT-05)

**Function: `async def _query_content_gaps(user_id: str) -> str`**
- Lazy import: `from services.lightrag_service import query_knowledge_graph`
- Call `await query_knowledge_graph(user_id=user_id, topic="content strategy gaps and unexplored topic domains", mode="hybrid")`
- Wrap in try/except — return empty string on any failure
- Log warning: `"LightRAG gap query failed for {user_id} (non-fatal): {e}"`

**Function: `async def _gather_user_context(user_id: str) -> dict`**
- Fetch persona from `db.persona_engines.find_one({"user_id": user_id})`
- Fetch last 10 content jobs: `db.content_jobs.find({"user_id": user_id, "status": {"$in": ["approved", "published"]}}).sort("created_at", -1).to_list(length=10)`
- Fetch performance data from those jobs (look for `performance_data` field if populated)
- Call `_query_content_gaps(user_id)` for LightRAG signal
- Return dict with keys: `persona`, `recent_content`, `performance_signals`, `knowledge_gaps`

**Function: `async def _build_synthesis_prompt(context: dict) -> str`**
- Build a prompt string that includes:
  - Persona card summary (archetype, content pillars, voice traits)
  - Recent content topics (last 10 jobs — platform, topic, hook used)
  - Performance signals (if any — engagement, reach from performance_data)
  - Knowledge graph gaps (from LightRAG, if available)
  - Suppressed topics list (so LLM avoids them)
- If knowledge_gaps is empty, note "No knowledge graph data available — focus on persona and recent content history"
- If performance_signals is empty, note "No performance data available yet — focus on content gaps and persona strengths"

**Function: `async def _synthesize_recommendations(user_id: str, context: dict) -> List[dict]`**
- Check `anthropic_available()` — return empty list if not
- Create LlmChat with `chat_constructor_key()`, session_id `f"strategist-{uuid.uuid4().hex[:8]}"`, system_message=STRATEGIST_SYSTEM_PROMPT
- Call `.with_model("anthropic", "claude-sonnet-4-20250514")`
- Wrap `chat.send_message(UserMessage(text=_build_synthesis_prompt(context)))` in `asyncio.wait_for(timeout=settings.strategist.synthesis_timeout)`
- Parse response with `json.loads(_clean_json(response))`
- Validate each item has required keys (topic, hook_options, platform, why_now, signal_source, generate_payload)
- Filter out any item missing required keys — log warning for each
- Return the parsed list

**Function: `def _build_generate_payload(rec: dict) -> dict`**
- Takes a raw LLM recommendation dict and returns a dict matching ContentCreateRequest schema exactly:
  ```python
  return {
      "platform": rec.get("platform", "linkedin"),
      "content_type": rec.get("generate_payload", {}).get("content_type", "post"),
      "raw_input": rec.get("generate_payload", {}).get("raw_input", rec.get("topic", "")),
  }
  ```
- This ensures the payload will NOT 422 when Phase 14 POSTs it to /api/content/generate (STRAT-07, Pitfall 4)

**Function: `async def run_strategist_for_user(user_id: str) -> dict`**
- Get cadence state. If `halved_rate` is True, max_cards = max_cards_per_day // 2 (minimum 1). Else max_cards = max_cards_per_day.
- Gather user context via `_gather_user_context(user_id)`
- Add suppressed topics to context (query `db.strategy_recommendations` for dismissed topics in last 14 days for this user, extract topic strings)
- Call `_synthesize_recommendations(user_id, context)`
- For each recommendation (up to max_cards):
  1. Normalize topic to lowercase
  2. Check `_is_topic_suppressed(user_id, topic)` — skip if suppressed
  3. Call `_atomic_claim_card_slot(user_id)` — break if returns False (cap reached)
  4. Build document:
     ```python
     {
         "recommendation_id": f"strat_{uuid.uuid4().hex[:12]}",
         "user_id": user_id,
         "status": "pending_approval",  # STRAT-02: ALWAYS this value
         "topic": topic,
         "hook_options": rec.get("hook_options", []),
         "platform": rec.get("platform", "linkedin"),
         "why_now": rec.get("why_now", ""),  # STRAT-03
         "signal_source": rec.get("signal_source", "persona"),
         "generate_payload": _build_generate_payload(rec),  # STRAT-07
         "created_at": datetime.now(timezone.utc),
         "expires_at": None,
         "dismissed_at": None,
         "dismissed_reason": None,
     }
     ```
  5. Insert into `db.strategy_recommendations`
- Update `db.strategist_state` with `last_run_at: datetime.now(timezone.utc)`
- Return `{"user_id": user_id, "cards_written": count, "skipped_suppressed": suppressed_count}`

**Function: `async def run_strategist_for_all_users() -> dict`**
- Call `_get_eligible_users()`
- For each user, call `run_strategist_for_user(user_id)` in a loop (NOT parallel — sequential to avoid LLM rate limits)
- Wrap each user call in try/except — log error and continue to next user on failure
- Collect results
- Return `{"total_users": len(users), "processed": processed_count, "total_cards": total_cards, "errors": error_count, "affected_user_ids": [list of user_ids that got cards]}`

**Function: `async def handle_dismissal(user_id: str, recommendation_id: str, reason: Optional[str] = None) -> dict`**
- Mark card dismissed in `db.strategy_recommendations`:
  ```python
  rec = await db.strategy_recommendations.find_one_and_update(
      {"recommendation_id": recommendation_id, "user_id": user_id, "status": "pending_approval"},
      {"$set": {"status": "dismissed", "dismissed_at": datetime.now(timezone.utc), "dismissed_reason": reason}},
      return_document=True,
  )
  ```
- If rec is None, return `{"error": "not_found"}`
- Update `db.strategist_state` — increment `consecutive_dismissals`:
  ```python
  state = await db.strategist_state.find_one_and_update(
      {"user_id": user_id},
      {"$inc": {"consecutive_dismissals": 1}},
      upsert=True,
      return_document=True,
  )
  ```
- Check if `consecutive_dismissals >= settings.strategist.consecutive_dismissal_threshold` AND `halved_rate` is not already True:
  - If so, set `halved_rate: True` and `needs_calibration_prompt: True` in strategist_state
- Return `{"dismissed": True, "topic_suppressed_until": (now + timedelta(days=settings.strategist.suppression_days)).isoformat(), "needs_calibration_prompt": bool}`

**Function: `async def handle_approval(user_id: str, recommendation_id: str) -> dict`**
- Mark card approved in `db.strategy_recommendations`:
  ```python
  rec = await db.strategy_recommendations.find_one_and_update(
      {"recommendation_id": recommendation_id, "user_id": user_id, "status": "pending_approval"},
      {"$set": {"status": "approved", "approved_at": datetime.now(timezone.utc)}},
      return_document=True,
  )
  ```
- If rec is None, return `{"error": "not_found"}`
- Reset consecutive dismissals in `db.strategist_state`:
  ```python
  await db.strategist_state.update_one(
      {"user_id": user_id},
      {"$set": {"consecutive_dismissals": 0, "halved_rate": False, "needs_calibration_prompt": False}},
  )
  ```
- Return `{"approved": True, "generate_payload": rec.get("generate_payload", {})}` (STRAT-07: returns the payload for one-click generation)
