    **1. Add ObsidianConfig to config.py (per OBS-03):**

    Add after StrategistConfig (before RemotionConfig):
    ```python
    @dataclass
    class ObsidianConfig:
        """Obsidian REST API configuration (Local REST API plugin).

        Per-user overrides stored in db.users.obsidian_config take precedence.
        These env vars serve as global fallback defaults.
        """
        base_url: str = field(default_factory=lambda: os.environ.get('OBSIDIAN_BASE_URL', ''))
        api_key: Optional[str] = field(default_factory=lambda: os.environ.get('OBSIDIAN_API_KEY'))

        def is_configured(self) -> bool:
            """Check if global Obsidian fallback is configured."""
            return bool(self.base_url and self.base_url.startswith('http'))
    ```

    Add `obsidian: ObsidianConfig = field(default_factory=ObsidianConfig)` to the Settings dataclass, between `strategist` and `remotion` fields (alphabetical order is not used — match existing ordering pattern).

    **2. Create backend/services/obsidian_service.py:**

    Module-level docstring explaining this wraps the Obsidian Local REST API plugin (https://github.com/coddingtonbear/obsidian-local-rest-api).

    Implement these functions:

    `_validate_vault_path(vault_path: str) -> str`:
    - Reject empty string
    - Reject absolute paths (starts with `/` or `\` or contains `:`)
    - Use `pathlib.PurePosixPath(vault_path).parts` to check no component is `..`
    - Also resolve via `PurePosixPath(vault_path)` and check `.parts` does not contain `..` after resolution
    - On violation: `raise ValueError(f"Path traversal blocked: {vault_path!r}")` and log at ERROR level
    - Return the cleaned path string on success

    `_get_user_obsidian_config(user_id: str) -> Optional[dict]`:
    - Query `db.users.find_one({"user_id": user_id}, {"obsidian_config": 1})`
    - If user has `obsidian_config` with `base_url` set, return it
    - Otherwise return None (caller falls back to global config)

    `_decrypt_obsidian_api_key(encrypted_key: str) -> str`:
    - Use the same Fernet pattern as routes/platforms.py — import `Fernet` from `cryptography.fernet`
    - Use `settings.security.fernet_key` as the encryption key
    - Decrypt and return the plaintext API key
    - On failure, log warning and return empty string

    `_resolve_config(user_id: Optional[str] = None) -> tuple[str, str, str]`:
    - Returns `(base_url, api_key, vault_path)` resolved from per-user config or global fallback
    - If user_id provided, call `_get_user_obsidian_config(user_id)`
    - Per-user config shape: `{"base_url": str, "api_key_encrypted": str, "vault_path": str, "enabled": bool}`
    - If per-user found and `enabled=True`: use per-user base_url, decrypt api_key_encrypted, use per-user vault_path
    - If per-user not found or not enabled: use `settings.obsidian.base_url`, `settings.obsidian.api_key`, vault_path=""
    - Validate vault_path via `_validate_vault_path()` if non-empty

    `is_configured(user_id: Optional[str] = None) -> bool`:
    - Returns True if either per-user or global Obsidian config has a valid base_url

    `async def search_vault(topic: str, user_id: Optional[str] = None, max_results: int = 5) -> dict`:
    - Resolve config via `_resolve_config(user_id)`
    - If not configured, return `{"findings": "", "vault_sources": [], "sources_found": 0}`
    - Build search URL: `{base_url}/search/simple/` with query param `query={topic}`
    - If vault_path is set, add `contextLength=200` and filter results to those whose `filename` starts with `vault_path/`
    - Headers: `{"Authorization": f"Bearer {api_key}", "Accept": "application/json"}`
    - Use `httpx.AsyncClient(timeout=10.0)` — wrap in try/except, return empty on ANY failure (OBS-04)
    - Parse response JSON — expect list of `{"filename": str, "matches": [{"match": {"content": str}}]}`
    - Build findings string: for each result, format as `"[Vault: {filename}] {snippet}"`
    - Return `{"findings": findings_str, "vault_sources": [{"title": filename, "snippet": snippet}], "sources_found": len(results)}`

    `async def get_recent_notes(user_id: str, hours: int = 24, max_results: int = 10) -> list[dict]`:
    - Resolve config via `_resolve_config(user_id)`
    - If not configured, return `[]`
    - Use `{base_url}/vault/` endpoint to list files (GET request)
    - Filter by vault_path prefix if set
    - For each file, check `{base_url}/vault/{filename}` metadata for modified time within `hours`
    - Return list of `{"title": str, "path": str, "modified": str}` for recent notes
    - Wrap everything in try/except, return `[]` on failure (OBS-04)
    - NOTE: If the Obsidian REST API does not expose per-file modified timestamps via listing, use an alternative approach: GET `/periodic/` endpoint if available, or use `/search/simple/` with a date-based query, or simply list vault files and use the response metadata. Document the approach in a code comment. The key contract is: return recently-modified note titles, or empty list if unavailable.
